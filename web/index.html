可信架构 · 六道防线
路由 / 算钱 / 引用 / 数字 / 案号 / 拒答，全部在代码层把关

LLM 只做两件事：抽取要素与依据检索到的条文生成。 是否动钱、引用真不真、数字从哪来、能不能答——全由确定性代码裁决。 一个回答要穿过下面六道防线，任一不过即拒答、转人工律师。

  1. 一
    路由决策
    先判该走哪条路——算钱、命中律师审核词条、检索生成，还是直接拒答。
  2. 二
    金额计算
    涉钱永远走确定性代码 calculators.py，模型不碰一个数字。
  3. 三
    引用校验
    所引法条必须真实存在，且落在本次检索到的条文范围内，越界即拦。
  4. 四
    数字溯源
    回答里每个数字，都要能追回条文、案例或你提供的事实，追不到即拦。
  5. 五
    案号校验
    答案中出现的案号，必须在库内案例里可查，杜绝编造判例。
  6. 六
    拒答闸门
    检索置信度不足、或算不准时，宁可拒答并转人工律师，不硬猜。
✓ 六关全过 → 输出回答结论、法条、金额逐项可核
✕ 任一不过 → 转人工律师不编造、不硬猜，留痕转介
